# Project Evidence

This directory is reserved for evidence supporting project claims, gates, reviews, benchmarks, compatibility results, security reviews, and launch decisions.

## Evidence classes

- protocol interoperability outputs;
- deterministic test logs;
- independent implementation comparison;
- security and privacy reviews;
- demo comprehension findings;
- SDK integration timing;
- connector sandbox records with secrets removed;
- site QA and accessibility reports;
- SEO/GEO audits;
- commercial experiments and anonymized findings;
- release and incident evidence.

## Rules

- Never commit production secrets, private keys, personal data, payment data, or confidential customer content.
- Evidence must identify date, version, command or method, expected result, actual result, and reviewer when applicable.
- Marketing claims must reference the smallest sufficient evidence.
- Screenshots are supporting evidence, not a substitute for machine-verifiable outputs.
