# Agentic Commerce and In-Chat Actions

## Definition

Agentic commerce and agentic actions move assistants from recommendation to execution: purchasing, booking, paying, sending, modifying, cancelling, refunding, and delegating directly from a chat or autonomous workflow.

## TimeProofs position

```text
User mandate
→ assistant or agent
→ action/commerce/payment protocol
→ receiving service
→ system of record
→ settlement or final outcome
→ TimeProofs evidence and assurance
```

TimeProofs does not replace the protocols in the path. It records and correlates what each layer can attest.

## Evidence questions for every protocol

- What exact user mandate or confirmation exists?
- What action was dispatched?
- What did the receiver accept?
- Which identifier links the request to the business record?
- Which system is authoritative for the claimed outcome?
- Is the result final, pending, partial, expired, cancelled, or reversible?
- What later event could contradict the result?
- Which evidence can be independently verified?
- What private data must remain outside the bundle?

## Initial commerce lifecycle

```text
INTENT
→ AUTHORIZATION
→ CHECKOUT OR ACTION DISPATCH
→ MERCHANT ACCEPTANCE
→ PAYMENT AUTHORIZATION
→ ORDER OR BUSINESS EFFECT
→ FULFILLMENT OR SETTLEMENT
→ RETURN / REFUND / REVERSAL / DISPUTE
```

No single phase may be presented as final completion unless the domain rules define it as sufficient.
