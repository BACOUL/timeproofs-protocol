# Global SEO Plan

## Objective

Build durable topical authority around outcome assurance, evidence for AI actions, and agentic commerce accountability without producing doorway pages or unsupported claims.

## Initial topic clusters

### Category

- outcome assurance for AI agents;
- AI action evidence;
- prove what an AI agent did;
- AI agent audit trail versus business evidence;
- system-of-record verification for agents.

### Protocol and developer

- Evidence Bundle specification;
- signed agent action receipt;
- MCP action verification;
- A2A task outcome evidence;
- agent action canonicalization and signatures;
- receiver-attested agent actions.

### Agentic commerce

- ACP outcome verification;
- UCP evidence mapping;
- AP2 payment mandate versus settlement;
- agentic checkout audit trail;
- AI refund verification;
- order, fulfillment, and reversal evidence.

### Use cases

- refunds;
- appointments;
- email;
- purchases and returns;
- account changes;
- claims and approvals.

## Technical requirements

- one canonical URL per intent;
- clear information architecture and internal links;
- structured data that matches visible content;
- XML sitemaps and correct robots policy;
- hreflang only for real translations;
- excellent Core Web Vitals and 320px layouts;
- semantic HTML and accessible navigation;
- no hidden AI-only claims;
- source, author, date, version, and limitation metadata.

## Content quality gate

A page is publishable only when it adds unique first-party value through at least one of:

- working tool or verifier;
- specification;
- reproducible example;
- original compatibility mapping;
- evidence-backed comparison;
- tested implementation guide;
- primary-source research synthesis.
